## 📝 技術的な学び

### 1. Mailbox 通知システムの仕組み

```
1. inbox_write.py がファイルに書き込み
2. inbox_watcher が 5 秒ごとにチェック
3. 変更を検知したら tmux に表示
4. 即座にファイルをクリア（セキュリティ）
```

**重要なポイント**:
- ファイルが空なのは正常（セキュリティのため）
- 詳細は br Issue に記載（永続化）
- 通知は「br を見て」の合図

### 2. br (beads_rust) の運用

**分かったこと**:
- `[全員]` タイトルだと自分宛だと認識しない
- タイトルに宛名を付ける（例：`[gemini][vibe]`）
- 新規 Issue は目立つ

**今後の改善**:
- Issue 作成時に宛名を明記
- 完了条件をチェックボックスで明記
