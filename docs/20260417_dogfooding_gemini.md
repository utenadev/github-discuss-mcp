## 1. 発生していた問題の分析

MCP サーバー（`github-discuss-mcp`）がリストに表示されながら `disconnected` となっていた原因は、以下の 2 点に集約されました。

### a) 環境変数（GITHUB_TOKEN）の読み込み不足
`cli.py` には設定されていた `.env` ファイルの読み込み処理（`load_dotenv()`）が、MCP サーバーのエントリーポイントである `main.py` に欠落していました。これにより、サーバー起動時に API トークンを取得できず、初期化に失敗していました。

### b) 実行ディレクトリへの依存性
従来の `load_dotenv()` はカレントディレクトリ（CWD）を基準にファイルを探すため、MCP クライアント（Cursor 等）がプロジェクトルート以外からサーバーを起動した場合に `.env` を見つけられないリスクがありました。

---

## 2. 実施した修正内容

以下の修正を `src/github_discuss_mcp/main.py` に適用しました。

1.  **絶対パスによる `.env` 読み込みの実装**:
    スクリプト自体の場所を基準にプロジェクトルートを特定し、絶対パスで `.env` を読み込むように変更しました。これにより、どのディレクトリから起動されても確実に設定が読み込まれます。
2.  **標準エラー出力（stderr）によるデバッグ支援**:
    トークンが未設定の場合に、エラー内容を `stderr` に出力する処理を追加しました。MCP クライアント側のログから原因を即座に特定可能になりました。
3.  **テストコードの更新**:
    追加されていたツール（`get_discussions`, `reply_to_discussion`）を反映し、`tests/test_mcp_server.py` の不整合を解消しました。
