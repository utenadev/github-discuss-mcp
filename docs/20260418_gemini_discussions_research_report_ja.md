# Gemini's GitHub Discussions 機能調査 & ギャップ分析レポート (2026-04-18)

## 📊 GitHub Discussions 機能調査 & ギャップ分析レポート

### 1. 現在の実装状況 (Implemented)
*   **ディスカッション作成**: `createDiscussion`
*   **カテゴリ一覧取得**: `discussionCategories`
*   **最新一覧取得**: `discussions(first: 10)`
*   **コメント追加**: `addDiscussionComment`
*   **ID 解決**: URL からの番号抽出による特定

### 2. 実装漏れ・未対応の主要機能 (Gaps)
| カテゴリ | 機能 (GraphQL フィールド/ミューテーション) | 詳細・メリット |
| :--- | :--- | :--- |
| **階層構造** | **`DiscussionComment.replies`** | コメントに対する「返信（Nested Replies）」の取得。スレッド形式の議論に必須。 |
| **階層構造** | **`addDiscussionComment(replyToId: ...)`** | 特定のコメントに対する「返信」の投稿。 |
| **状態管理** | **`markDiscussionCommentAsAnswer`** | Q&A カテゴリで「回答」としてマークする。AI が質問に答えた際にあるべき機能。 |
| **コンテンツ操作** | **`updateDiscussion` / `updateDiscussionComment`** | 誤字修正や AI による推敲・更新に必要。 |
| **コンテンツ操作** | **`deleteDiscussion` / `deleteDiscussionComment`** | テスト投稿の削除や不要な情報の整理。 |
| **メタデータ** | **`Reactions`** | 👍 や 🎉 などのリアクションの取得・追加。AI が感情を示す手段。 |
| **メタデータ** | **`Labels`** | Issue 同様、ラベルによる分類管理。 |
| **検索・発見** | **`Search` (v4)** | `search(query: "is:open label:bug ...", type: DISCUSSION)` による高度な検索。 |

### 3. 技術的に重要な発見
*   **返信の二段階構造**: ディスカッションには「コメント (Comment)」と、そのコメントに対する「返信 (Reply)」があります。現在の `reply_to_discussion` は「コメント」を追加するだけで、スレッド内の特定の意見に「返信」することはできません。
*   **検索の柔軟性**: `Repository.discussions` よりも、グローバルな `search` クエリを使うほうが、キーワード、著者、状態などを組み合わせた強力な探索が可能です。

### 💡 今後のロードマップ提案 (Phase 1 優先)

この調査結果を踏まえ、ドッグフーディングを「きちんと」進めるため、以下のフェーズで機能を強化します。

#### **フェーズ1: 議論の質の向上 (最優先)**
*   **コメントへの返信機能**: `reply_to_discussion` を拡張し、特定のコメントに対しても返信できるようにする。
*   **詳細取得の強化**: コメントだけでなく、その下の返信（Replies）も階層構造で取得できるようにする。

#### **フェーズ2: コンテンツ管理と Q&A 対応**
*   **更新・削除機能**: 投稿した内容を AI 自身が修正・整理できるようにする。
*   **Q&A 対応**: 回答のマーク機能。

#### **フェーズ3: 探索と通知**
*   **高度な検索**: キーワード検索ツールの追加。
*   **メンション通知**: 自分宛の言及を効率よく見つける仕組み。
